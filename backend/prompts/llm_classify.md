==== 四种演讲类型定义 ====

**1. 产品创新型 (innovation)**
核心特征：发现用户痛点或未被满足的需求 → 引入新技术/新设计/新算法来解决 → 用测试数据/原型/市场认可度证明效果 → 探讨跨部门复用价值
典型关键词：创新、新技术、新方案、用户需求、市场、原型、突破、跨界、迭代

**2. 问题解决型 (troubleshooting)**
核心特征：遭遇设计失效/质量问题/用户投诉 → 深入排查定位根本原因（Root Cause） → 采取针对性对策 → 提炼防错规则供复用
典型关键词：失效、投诉、排查、根本原因、Root Cause、对策、改善前后、防错、归零

**3. 降本增效型 (cost_reduction)**
核心特征：面临成本压力或降本目标 → 在保证（甚至提升）原有性能前提下压缩成本 → 替换新材料/简化结构/优化工艺 → 量化节省金额与性能达标证明
典型关键词：降本、成本、节省、材料替换、工艺优化、结构简化、工时、效率提升、性价比

**4. 方法工具改进型 (methodology)**
核心特征：日常开发/测试中某环节最耗时、最痛苦或最容易出错 → 旧工具/手工操作低效 → 开发自动化脚本/工具/新标准/新流程 → 量化效率提升并推广共享
典型关键词：自动化、工具、脚本、流程、方法、标准化、推广、开源共享、复用

==== 判断规则 ====
- 以PPT内容和演讲原文的主要叙事脉络为主要判断依据
- 如存在多种特征，选择最主要、最突出的那种类型
- 必须选择且只能选择上述4种之一
- type_key 必须严格为以下枚举值之一：innovation / troubleshooting / cost_reduction / methodology
